UNIVERSAL SERVICE AGGREGATOR PROJECT BRIEF v2.1
Purpose

Build a modern service aggregator website that helps users connect with independent local service providers.

The website must not appear to be a clone of previous projects.

Before Starting

Do not begin implementation until all available project information has been reviewed.

Wait for:

niche
company name
service list
user instructions
visual references
branding requirements

References always override default assumptions.

Design Rules

Follow user-provided references.

Do not invent visual direction when references exist.

Avoid:

repeating previous homepage layouts
repeating previous hero sections
repeating previous section order
repeating previous design systems

Each project must feel unique.

Content Rules

Content must be unique.

Do not reuse:

About sections
FAQ sections
CTA copy
testimonials
service descriptions

Do not simply replace company names in existing content.

Technical Stack

Use:

HTML
CSS
JavaScript

Avoid unnecessary frameworks unless requested.

Required Pages
Home
About
Services
Individual Service Pages
Contact
Privacy Policy
Terms of Service
Cookie Policy
Configuration

Store company-level information in:

js/site-config.js

Use configuration for:

company name
phone
email
address
service area
business hours

Do not hardcode reusable business information.

Service Pages

Service content must be static HTML.

Do not generate services from:

services-data.js

Do not use content hydration.

Content must be indexable without JavaScript.

Conversion Requirements

Include meaningful CTAs:

Hero
Service Sections
Service Pages
Contact Page

Recommended:

Call Now
Request Quote
Get Help Today
Tracking Safety

Never hardcode:

Google Analytics IDs
Google Tag Manager IDs
Meta Pixel IDs
TikTok Pixel IDs
Google Ads Conversion IDs

Only add tracking when provided.

Images

Use:

project-specific images
user-provided assets
generated images when necessary

Avoid:

placeholders
duplicated imagery across projects
broken paths

Prefer:

.webp
Responsive Requirements

Validate:

Desktop
Tablet
Mobile
360px

Requirements:

no horizontal scrolling
readable navigation
touch-friendly interactions
SEO Requirements

Required:

unique page titles
semantic headings
sitemap.xml
robots.txt
internal linking
alt text
Accessibility

Required:

form labels
aria attributes
visible focus states
keyboard accessibility where practical
Performance

Prioritize:

fast load speed
optimized assets
minimal dependencies

Avoid:

unnecessary libraries
oversized images
unused code
Final Review

Before completion verify:

references followed
content unique
no placeholder content
no broken links
no broken assets
responsive checks passed
SEO checks passed
accessibility checks passed
no console errors
no shared analytics IDs
website does not appear cloned from previous projects
